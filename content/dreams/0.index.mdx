# **DSA Rulebook: Arrays**

Arrays are one of the most fundamental data structures in problem-solving. To choose the correct approach, follow this structured decision-making guide.

---

<img src="/sample.avif" alt="Sample" width="400" className="centered" />

## **1. Is the array sorted?**

- **Yes** → Consider **Binary Search** (O(log n))
  - If the problem involves searching, find elements efficiently using **Binary Search**.
  - If modifications (insertion/deletion) are allowed, consider **Balanced BST** or **Ordered Map**.
  - If two indices need to be found (sum or difference condition), use **Two Pointers**.

---

## **2. Is the problem asking for a maximum/minimum element continuously?**

- **Yes** → Consider:
  - **Heap (Priority Queue)** → If you need to get the max/min dynamically.
  - **Monotonic Queue** → If the problem involves sliding window and max/min queries.

---

## **3. Is the problem about finding a subarray or substring with a condition?**

- **Yes** → Consider:
  - **Sliding Window** → If the window expands/shrinks dynamically.
  - **Two Pointers** → If the array needs partitioning based on a condition.
  - **Prefix Sum** → If cumulative sums are needed for range queries.
  - **Kadane’s Algorithm** → For the maximum sum subarray.

---

## **4. Are we searching for a specific element or checking for existence?**

- **Yes** → Consider:
  - **Hash Map / Hash Set** → Quick lookup (O(1) average).
  - **Binary Search** → If the array is sorted.

---

## **5. Are elements added or removed in a sliding window fashion?**

- **Yes** → Consider:
  - **Deque (Monotonic Queue)** → To maintain max/min values dynamically.
  - **Sliding Window with Hash Map** → To track unique elements dynamically.

---

## **6. Does the problem ask for the number of ways to do something?**

- **Yes** → Consider:
  - **Dynamic Programming (DP)** → If subproblems overlap.
  - **Backtracking** → If all combinations must be explored.

---

## **7. Is the problem about merging or partitioning arrays?**

- **Yes** → Consider:
  - **Two Pointers** → If arrays are sorted.
  - **Merge Sort** → Efficient merging (O(n log n)).
  - **Quick Sort** → Partitioning around a pivot.

---

## **8. Is the problem asking for distance between elements?**

- **Yes** → Consider:
  - **Stack / Monotonic Stack** → For nearest greater/smaller elements.
  - **Two Pointers** → For comparing distances.

---

## **9. Is the problem about prefix matching or searching substrings efficiently?**

- **Yes** → Consider:
  - **Trie** → For dictionary-based lookups.
  - **KMP / Rabin-Karp** → For fast substring search.

---

## **10. Is the problem checking if a sequence is valid (e.g., brackets)?**

- **Yes** → Consider:
  - **Stack** → When LIFO behavior is required.

---

## **Conclusion**

Understand the problem requirements and recognize patterns. Arrays are foundational to advanced structures like stacks, queues, and heaps.

## **Next Topic: Strings**

### **1. Is it about checking anagrams or matching characters?**

- **Yes** → Use:
  - **Sorting & Comparing**
  - **Hash Map / Frequency Count**

### **2. Is it about substring search?**

- **Yes** → Use:
  - **KMP / Rabin-Karp**
  - **Sliding Window / Rolling Hash**

### **3. Generating/validating a string sequence?**

- **Yes** → Use:
  - **Stack**
  - **Backtracking**

### **4. Longest substring/subsequence with a property?**

- **Yes** → Use:
  - **Sliding Window**
  - **Dynamic Programming (DP)**

### **5. Compressing or encoding a string?**

- **Yes** → Use:
  - **Run-Length Encoding**
  - **Huffman Encoding**

### **6. Word segmentation or breaking into words?**

- **Yes** → Use:
  - **Trie**
  - **Dynamic Programming (DP)**

### **7. Constructing strings from operations?**

- **Yes** → Use:
  - **Stack**
  - **Greedy Algorithms**
